# Deep mode — the perspective panel

An optional heavy pass for high-stakes sessions: 3–4 independent perspectives
examine the front-runners, then you synthesize. Its value is diversity — a
single mind (including yours) converges too early and defends its first good
idea. The panel exists to break that.

## When to recommend it

Recommend once, in phase 4 (Challenge), when at least one of:

- the decision is a one-way door (irreversible or costly to reverse),
- it commits significant money, years, or other people,
- the front-runners are close and the user is torn,
- everyone involved already agrees (consensus reached fast is a red flag).

Recommend with a one-line cost note: it spawns several agents / long passes
and burns noticeably more tokens than the normal flow. Run it ONLY on explicit
yes. After a no, never re-offer in the same session.

## Composing the panel

Pick 3–4 perspectives chosen for TENSION with each other, not coverage for its
own sake. Adapt to the topic; defaults:

**Business topics:**
- **The skeptic** — assumes the plan fails; argues the strongest case against
  each front-runner. Not a devil's advocate performing disagreement — a
  genuine "this is why it dies".
- **The other side** — the customer, partner, or counterparty. What does the
  deal look like from their chair? Where does it quietly screw them (they'll
  notice), or spoil them (unsustainable)?
- **The operator** — assumes the decision is made; what breaks in execution in
  the first 90 days? Which resource is actually the bottleneck?
- **The capital allocator** — compares the front-runners against doing nothing
  and against the best alternative use of the same money/time.

**Life topics:**
- **Future self** — ten years out, living with the consequences. What do they
  thank you for, what do they resent?
- **The affected** — partner, kids, close friends. Argues their interest
  honestly, including where it conflicts with the user's.
- **The stoic** — separates what's in the user's control from what isn't;
  strips status and audience-pleasing motives out of each option.
- **The opportunity cost** — argues for what the same time/money/energy would
  buy elsewhere.

## Running it

Each perspective gets the same brief: the frame (question, criteria,
constraints, verbatim), the front-runners, and relevant memory context. Each
must return exactly:

1. the strongest case FOR the option it favors,
2. the strongest case AGAINST the current favorite,
3. ONE question the user must answer before deciding,
4. its verdict in one sentence.

Run the perspectives as parallel subagents when an agent-spawning tool is
available. When it isn't, run them sequentially in-conversation — one
perspective per pass, fully in character, WITHOUT letting earlier passes bleed
into later ones (each pass argues fresh from the brief, not from the previous
pass's conclusions).

## Synthesis — where the value is

Do not average the panel into mush. Produce:

- **Disagreement map** — where perspectives clash, in one table; the clashes
  ARE the finding.
- **Convergent warnings** — anything two or more perspectives flagged
  independently gets promoted to a top risk.
- **The hinge** — the single assumption or unknown that, if it resolved one
  way or the other, would flip the decision. If the panel surfaced one, the
  next action is usually "resolve the hinge", not "decide now".
- **Your recommendation, updated** — say explicitly whether the panel changed
  it and why.

Present the user the panel's questions (item 3 from each perspective) one at a
time before final convergence — they are usually the best questions of the
whole session.
